# まごころ 設計原則 (ver1.2)

> 以降のすべての実装日（D2-）で守る判断基準。迷ったらここに戻る。
> 法務文言の確定は専門家確認後（D24 / W23-24）。本書はそれまでのメモ。

## 1. 役割の限界（非医療・非緊急通報）
- まごころは医療行為・診断・治療助言をしない。
- 緊急通報（119 など）の代替にはならない。緊急時は本人・家族・公的窓口へ誘導する。

## 2. ガードレール4段
危険・不適切な入出力に対し、段階的に対応する：
1. **検知** — リスク（医療・自傷・緊急・差別 等）を検出する。
2. **抑止** — 不適切な応答を生成しない／止める。
3. **言い換え** — 安全で穏やかな表現に置き換えて返す。
4. **人へ** — 自動応答で完結させず、家族・ケアマネ等の人につなぐ。

## 3. ハルシネ対策（不確実は「わからない」）
- 確証のない情報は断定せず、「わからない」と正直に答える。
- 事実不明な点を、もっともらしく作話しない。

## 4. 記憶の取り扱い（本人が主導できる）
利用者本人が自分の記憶を：
- **確認** できる（何を覚えているか見える）
- **修正** できる
- **上書き** できる
- **削除** できる

## 5. ケアマネ通知は三者同意時のみ
- 本人・家族・ケアマネの**三者同意**が取れた場合に限り、ケアマネへ通知する。
- 同意がない情報共有はしない。

## 6. 見守り通知の二重化
- 通知経路を二重化する：**LINE Push + メール**。
- **未読再通知**を行う（届かない／読まれないリスクに備える）。
